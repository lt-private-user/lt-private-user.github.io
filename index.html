Check-Ins

#
  • Es sollte min. einer aus der Gruppe bei den wöchentlichen "Check-Ins" anwesend sein.
  • Nominierte Gruppen müssen (vollständig) anwesend sein.
  • Check-In Inhalte können in der finalen Präsentation wiederverwendet werden.

Themen fürs SteerCo

#
  • Magisches Dreieck (Budget, Termin, Funktion) im Termin behandeln.
  • Projektstrukturplan/Backlog definieren + präsentieren.
  • Risikomanagement vorstellen, sofern notwendig (empfohlen).
  • Kommunikationsplan (wenn man möchte).

Fertigstellungstermin / Zieltermin

#
  • Vorschlag Zieltermin(e):
  • Podcast: 25.05.2026 (1 Woche vor Abgabe).
  • Präsentation: 30.05.2026 (1 Tag vor Abgabe).

Projekt – Allgemeine Informationen

#
  • Entwicklungsansatz frei wählbar (in der Präsentation aufgreifen + erklären der Vorgehensweise) → so, dass es für uns passt; alles andere weglassen (nur Dinge, welche essenziell für den Projekterfolg sind).
  • Weglassen muss begründet werden (können).
  • Nach Festlegung auch daran halten (eher: so etwas wie SCRUM; statt: SCRUM).
  • Kosten/Zeit schätzen.
  • Ziele definieren (SMART, DoD).
  • Meilensteine definieren → ggf. Meilenstein-Trend-Analyse.
  • Ein (nicht disruptiver) Änderungswunsch wird kommen → Change Request.
  • Änderungswunsch ansprechen, überprüfen, Ergebnisse präsentieren (Was wenn ja, was wenn nein).
  • Dokumentationspflicht des Projektmanagers.
  • Dokumentation erstellen.

Podcast

#
  • Kein Transkript notwendig (nicht empfohlen).

Präsentation

#
  • Fragen der Dozenten zu Theorie/Grundlage.

Change Request

#
  • Jede Gruppe muss auf die nächste Gruppe verweisen → Kette / zusammenhängende Podcast-Folgen.
  • Reihenfolge am besten heute noch klären.
